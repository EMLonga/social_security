# 社区安全预警系统 PRD（产品需求文档）

## 1. 产品背景
美国社区公共安全数据分散在不同开放平台，普通用户难以在短时间内判断“我所在社区当前是否安全”。本系统通过自动抓取、分类、社区归属和可视化，降低信息获取门槛。

## 2. 产品目标
- 聚合公开事件数据，形成统一可检索事件库。
- 让用户在首页 10 秒内获得“近期风险概况”。
- 让用户在 3 次点击内查看目标社区的报告与近期事件。
- 提供运营后台保障内容质量、分类质量和数据更新效率。

## 3. 用户角色
- 游客：查看事件与社区基础信息。
- 注册用户：点赞、收藏、评论、关注社区、导出详情。
- 管理员：管理事件/社区/评论/用户，触发爬虫、调整站点配置。

## 4. 关键场景

### 4.1 快速感知风险
用户打开首页，通过地图 + 热门事件 + 社区排名了解近期风险重点。

### 4.2 深入查看单个事件
用户进入事件详情，查看时间地点、来源、描述、评论和关联社区。

### 4.3 追踪社区安全趋势
用户在社区详情查看评分、趋势、报告、近期事件分布及类型构成图。

### 4.4 运维与治理
管理员在后台处理不当评论、维护敏感词、修正异常事件、手动触发抓取。

## 5. 功能需求

### 5.1 首页
- 地图事件点位展示
- 事件类型筛选（含地震）
- 时间范围筛选（7/30/90 天）
- 热门事件列表（综合互动热度）
- 安全社区排行与统计概览

### 5.2 事件列表
- 多条件筛选：关键词、类型（多选）、社区、时间范围、风险等级、排序
- 列表分页（页码 + 每页条数）
- 点赞/收藏快捷操作

### 5.3 事件详情
- 事件地图定位
- 事件元信息与来源链接
- 详细描述
- 评论发布与删除
- 点赞、收藏、分享、导出

### 5.4 社区列表（社区报告）
- 报告卡片展示
- 分页浏览
- 点击进入社区详情

### 5.5 社区详情
- 社区评分、等级、趋势
- 人口、面积、30 天事件数、主导类型
- 地图展示社区与事件分布
- 图表（类型占比 + 时间趋势）
- 自动生成报告（高风险时段/地点/建议/对比）
- 近期事件分页

### 5.6 账户系统
- 注册、登录、验证码
- 找回密码（邮件验证码）
- 个人中心（资料、收藏、关注、评论）

### 5.7 管理后台
- 仪表盘统计
- 事件管理（增删改查）
- 社区管理（增删改查）
- 评论管理（标记/解除/删除）
- 用户管理（角色/状态/删除）
- 爬虫状态、触发与停止
- 站点配置与敏感词管理

### 5.8 AI 助手
- 基于当前页面上下文回答问题
- 支持中英文
- 不应虚构系统不存在功能

## 6. 业务规则

### 6.1 事件分类
- 类别：`theft/shooting/fire/security/fraud/earthquake/other`
- 未命中规则时归 `other`。

### 6.2 社区归属
- 优先按地址解析 `state + city` 精确匹配。
- 无匹配时允许创建新社区并长期复用。
- 事件必须尽可能归属到某个社区，减少孤立数据。

### 6.3 热门事件
- 热门由点赞数与评论数综合决定，并按时间降序补充“最近性”。

### 6.4 地震影响控制
- 地震可独立分类。
- 低/中危地震对社区评分影响应弱于严重治安类事件。
- “全部类型”展示需避免单一地震数据挤占视图。

## 7. 数据与算法需求

### 7.1 数据源
- 城市开放数据（治安类）
- 地震与灾害类（USGS/NWS/CAL FIRE/NASA 等）

### 7.2 评分
- 输入：事件类型、风险等级、时间衰减、未来预警
- 输出：`safety_score`, `safety_level`, `trend`

### 7.3 报告生成
- 从社区事件集中提取热点时段、热点地点和建议文本
- 自动补全缺失报告

## 8. 非功能需求
- 可用性：前后端启动后可稳定浏览和检索
- 性能：分页查询支持千级事件量
- 可维护性：路由、服务、模型分层清晰
- 可扩展性：可增新数据源和新分类规则

## 9. 验收标准（当前版本）
- 首页可按类型与时间刷新地图和热门事件
- 事件列表筛选、分页、排序生效
- 社区报告列表支持分页
- 事件详情评论与互动可用
- 社区详情图表与近期事件分页可用
- 后台可触发抓取并查看状态

## 10. 后续规划
- 引入更强地理编码，提高州/城市解析准确率
- 热门模型加入时间衰减热度
- 翻译词典持续完善，覆盖长尾城市名
- 加入更完整的自动化测试链路
